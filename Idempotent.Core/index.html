Idempotent.Core is the foundational library that implements the core idempotency model, workflows, and persistence abstractions.
It is framework-agnostic and contains no HTTP or ASP.NET dependencies.

This package defines what idempotency is, how transactions behave, and how stores must implement consistency, locking, eviction, and validation.

This package is not intended to be consumed directly by application code.

Idempotent Transaction is uniquely identified by a composite key derived from:

  • Service name
  • HTTP-like method
  • Request path
  • Optional query string
  • Client-provided idempotent key

Fingerprinting prevents accidental replay of logically different requests under the same idempotency key. Requests are fingerprinted using:

  • HTTP method
  • Path + query
  • Content type
  • Request body stream

SemaphoreSlim is used for concurrency and provides:

  • Per-transaction locking using
  • Lock eviction to prevent unbounded memory growth
  • Store eviction and lock eviction run independently on intervals
